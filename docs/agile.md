Agile, Scrum y Kanban — Investigación
¿Qué es Agile y cuál es su objetivo?
Agile es una metodología de desarrollo de software basada en iteraciones cortas, colaboración constante y adaptación rápida a los cambios.
Su objetivo principal es entregar valor de forma continua, evitando procesos rígidos y largos que retrasan el desarrollo.

Agile promueve:

Entregas frecuentes

Comunicación constante

Flexibilidad ante cambios

Retroalimentación continua

Mejora constante del proceso

¿Qué es Scrum?
Scrum es un marco de trabajo dentro de Agile que organiza el desarrollo en ciclos llamados sprints, normalmente de 1 a 4 semanas.

Roles principales
Product Owner (PO): define qué se debe construir y prioriza el backlog.

Scrum Master (SM): facilita el proceso, elimina bloqueos y asegura que se sigan las reglas de Scrum.

Development Team: equipo que desarrolla el producto.

Eventos de Scrum
Sprint Planning: planificación del sprint.

Daily Scrum: reunión diaria de 15 minutos.

Sprint Review: revisión del trabajo completado.

Sprint Retrospective: análisis de mejoras.

Artefactos
Product Backlog: lista priorizada de tareas.

Sprint Backlog: tareas seleccionadas para el sprint actual.

Incremento: resultado del sprint.

¿Qué es Kanban?
Kanban es un sistema visual para gestionar tareas mediante un tablero dividido en columnas que representan el flujo de trabajo.

Ejemplo típico:

Backlog

Todo

In Progress

Review

Done

Kanban se centra en:

Visualizar el trabajo

Limitar tareas en progreso (WIP)

Mantener un flujo continuo

Entregar sin ciclos fijos

No tiene roles obligatorios ni ceremonias formales.

Diferencias entre Scrum y Kanban
Aspecto	Scrum	Kanban
Ciclos	Sprints fijos	Flujo continuo
Roles	PO, SM, Dev Team	No requiere roles
Cambios	No se cambian tareas durante el sprint	Cambios permitidos en cualquier momento
Reuniones	Varias ceremonias	No obligatorias
Enfoque	Iteraciones	Flujo constante
Planificación	Por sprint	Continua


¿Cuándo usar Scrum?
Cuando el proyecto necesita estructura y planificación por ciclos.

Cuando el equipo es mediano o grande.

Cuando se requiere claridad en roles y responsabilidades.

Cuando se trabaja por objetivos definidos.

¿Cuándo usar Kanban?
Equipos pequeños o de soporte técnico.

Proyectos con tareas impredecibles.

Cuando se necesita flexibilidad total.

Cuando se quiere evitar reuniones constantes.

Conclusión
Scrum y Kanban son metodologías ágiles que ayudan a organizar el trabajo, pero cada una sirve para situaciones distintas.
Para un proyecto como SupportFlow, Kanban es ideal porque permite gestionar tareas de forma flexible y visual, sin ciclos rígidos.
