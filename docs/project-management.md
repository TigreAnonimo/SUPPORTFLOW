Gestión del Proyecto — SupportFlow
Metodología utilizada
Para este proyecto se utiliza Kanban, ya que es ideal para:

proyectos con tareas cambiantes

flujos continuos de trabajo

equipos pequeños

evitar ciclos rígidos como los sprints

El tablero permite visualizar el estado de cada tarea y moverlas según el progreso.

Estructura del tablero
El tablero contiene las siguientes columnas:

Backlog: ideas y tareas pendientes de priorizar

Todo: tareas listas para empezar

In Progress: tareas en desarrollo

Review: tareas en revisión

Done: tareas completadas

Tarjetas del proyecto
Cada tarjeta representa una funcionalidad o tarea técnica.
Ejemplos:

Crear estructura del proyecto

Configurar Tailwind

Crear API Client

Crear backend Express

Crear componentes

Documentación

Despliegue

Cada tarjeta incluye subtareas técnicas para facilitar el desarrollo.

Flujo de trabajo
Las tareas comienzan en Backlog

Se mueven a Todo cuando están listas

Se pasan a In Progress cuando se están desarrollando

Luego a Review para revisión final

Finalmente a Done cuando están completadasl
